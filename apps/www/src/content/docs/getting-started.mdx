Run Cliparr in Docker, then open the app and connect Plex, Jellyfin, or a local video. Mount `/data` before connecting media servers so settings and encrypted credentials survive updates.

<CommandBlock variants={dockerRunCommandVariants} label="Docker quick start" />

<p>{dockerLinuxContainerNote}</p>

Open `http://localhost:7171`, connect Plex or Jellyfin, or choose **Open Video** for a local file.

<Callout title="Stable APP_KEY required" tone="warning">
  Use a stable random `APP_KEY` at least 32 characters long. If it changes
  later, Cliparr cannot decrypt the stored provider credentials and you will
  need to reconnect sources.
</Callout>

<Callout title="Use HTTPS for editing" tone="warning">
  Cliparr uses browser WebCodecs in the editor. Use an HTTPS reverse proxy for
  remote access, or open the app through `localhost` or `127.0.0.1` while
  testing locally. See [Browser support](/docs/browser-support) for the current
  feature support snapshot.
</Callout>
